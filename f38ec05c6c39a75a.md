# LinkWiki

一个面向技术研究者与内容聚合者的结构化外链资源管理与导航系统。LinkWiki 定位于将分散在网络各处的深度技术文章、百科条目与专业资料，以统一索引和分类体系进行组织，帮助用户在海量信息中快速定位高价值内容。本系统适用于个人知识库构建、团队技术文档沉淀以及公开资源站点的快速搭建，尤其适合需要定期整理和呈现大量外链引用资料的运营场景。

LinkWiki 不提供内容存储或代理转发，而是作为元数据索引层与导航界面，通过标准化条目格式和分类标签，将原始来源链接进行有序编排。其核心价值在于降低外链资源的维护成本，提升信息检索效率，并提供可定制的展示框架。

## 功能概览

**结构化条目索引**：每条资源按唯一标识符、标题摘要、来源域名和原始 URL 进行规范化登记，支持按时间、分类、热度等多维度排序。

**分类标签体系**：内置三级分类标签系统，允许为每个条目分配多个标签，实现跨目录的多维筛选与关联推荐。

**快速全文检索**：基于标题和摘要字段的轻量级全文搜索，支持关键词高亮和模糊匹配，响应时间控制在 200 毫秒以内。

**外部链接健康检查**：定期对已收录的 URL 进行可达性探测，自动标记失效链接并生成报告，保障资源列表的有效性。

**批量导入与导出**：支持 CSV、JSON 格式的批量条目导入，以及完整索引数据的导出，便于迁移和备份。

**访问统计与热度排行**：记录每个条目的点击次数和最近访问时间，自动生成周榜和月榜，辅助识别热门资源。

**自定义主题与布局**：提供多套前端主题切换能力，允许管理员调整列表密度、卡片样式和侧边栏组件位置。

**开放 API 接口**：提供 RESTful API 用于第三方系统的条目查询、分类拉取和统计上报，便于集成到现有工作流。

## 应用场景

技术团队内部知识库维护：开发团队可将日常调研中发现的优质技术博客、官方文档和解决方案文章，通过 LinkWiki 统一收录并分配类别标签，新成员入职时可快速浏览团队积累的推荐阅读列表，缩短学习曲线。

个人研究者的阅读清单管理：从事计算机科学、数据科学或工程研究的个人，可将大量分散的预印本、技术报告和实验笔记链接集中存放，配合分类标签和检索功能，在撰写文献综述或进行实验回溯时高效调取相关资料。

公开技术资源导航站点运营：社区或开源组织可基于 LinkWiki 搭建面向特定领域（如操作系统内核、分布式系统、机器学习框架）的外部资源导航页，为访问者提供经过筛选和分类的高质量链接索引，替代传统书签分享方式。

内容审核与质量回溯：编辑团队可利用 LinkWiki 的条目备注和版本记录功能，对收录的每一篇外部文章进行简评和质量标记，在后续内容更新或合规审查时快速定位需要复核的条目。

## 快速开始

以下指令适用于 Linux 与 macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkwiki.git
cd linkwiki

# 安装依赖（使用 pip 与虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化数据库并启动开发服务
python manage.py migrate
python manage.py loaddata initial_categories.json
python manage.py runserver 0.0.0.0:8000
```

访问 http://localhost:8000 即可进入 LinkWiki 首页，默认管理员账号为 admin，密码为 linkwiki2024，首次登录后请立即修改。

## 贡献指南

1. 在 GitHub 上 Fork 本仓库，并基于 main 分支创建以 feature/ 或 fix/ 为前缀的命名分支，例如 feature/batch-import-optimize。所有开发应在分支内完成，避免直接向 main 提交。

2. 新增功能或修复缺陷时，请同步编写或更新相应的单元测试，测试文件放置于各应用下的 tests/ 目录。确保现有测试全部通过，且新代码的测试覆盖率不低于 80%。

3. 提交代码前执行预提交检查，包括 flake8 语法检查、black 代码格式化以及 isort 导入排序。项目根目录提供 .pre-commit-config.yaml，安装 pre-commit 后运行 pre-commit install 即可自动启用。

4. 针对资源列表的更新（包括新增、删除或修改 URL），需同时维护对应的分类标签和摘要描述，保证数据完整性。若涉及外部链接变更，请在 PR 描述中注明是否已执行健康检查。

5. 提交 Pull Request 时请参照 PR 模板填写变更类型、影响范围、测试情况和相关 Issue 编号。至少需要一位项目维护者审核通过后方可合并。

## 常见问题

Q: 如何批量导入大量外链而不逐条手动录入？

A: 使用 scripts/import_csv.py 工具，按照示例格式准备包含 title、url、category、tags 列的 CSV 文件，执行 python scripts/import_csv.py --file your_data.csv 即可完成批量导入。导入前建议先使用 --dry-run 参数进行预检查。

Q: 链接健康检查报告显示大量超时，如何调整探测参数？

A: 健康检查的超时和重试阈值可在 apps/entries/url_health.py 顶部的 TIMEOUT 和 RETRY 常量中调整。默认超时为 5 秒，重试 2 次。若目标站点普遍响应较慢，可适当增加超时至 10 秒，并减少并发数量以避免被反爬策略拦截。

Q: 能否将 LinkWiki 部署为静态站点生成器模式，以降低服务器要求？

A: 当前版本依赖 Django 服务端动态渲染，不支持完全静态导出。但可通过配置页面缓存（Redis 缓存后端）和 CDN 加速静态资源来减轻服务器负载。对于纯静态需求，建议使用第三方爬取工具将前端页面保存为 HTML，但会失去检索和统计功能。
