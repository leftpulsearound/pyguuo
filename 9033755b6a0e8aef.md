# KnowledgeLink 聚合站

KnowledgeLink 是一个面向技术研究者、开源贡献者和内容策展人的外链资源聚合平台。该项目通过结构化梳理与人工筛选，将散落于互联网各处的百科条目、技术文档与知识论述整合为可检索、可分类、可追溯的持久化链接库。项目定位为“知识路由”而非“知识容器”，帮助用户在信息过载的环境中快速定位高价值原文。

本项目服务于以下人群：需要批量查阅垂直领域百科资料的分析师、维护个人知识库的开发者、搭建站内搜索或推荐系统的算法工程师，以及希望减少重复检索时间的日常信息消费者。KnowledgeLink 不存储任何第三方内容，仅提供稳定且可验证的 URL 引用层，确保原始来源的版权归属与内容完整性。

## 功能概览

**批量外链导入**：支持从纯文本、CSV 或 JSON 格式批量导入 URL 列表，自动去重并校验可访问性，减少人工整理成本。

**智能分类标签**：基于 URL 路径特征与来源域名自动生成分类标签，用户可手动调整标签体系，支持多级标签嵌套与重命名。

**链接状态监控**：定时检测已收录链接的 HTTP 状态码与响应时间，标记失效链接并生成变更报告，保障引用链路的健康度。

**全文元数据提取**：对收录页面自动提取标题、摘要、关键词与发布时间，支持自定义 XPath 或 CSS 选择器规则，适配不同站点结构。

**检索与过滤**：提供基于标题、标签、域名、时间范围的多条件组合检索，支持正则表达式高级搜索，结果可按相关度或更新时间排序。

**导出与集成**：支持将筛选后的链接列表导出为 Markdown、JSON、RSS 或 OPML 格式，便于嵌入个人博客、wiki 或自动化工作流。

**用户自定义视图**：允许用户创建私有或公开的链接集合视图，每个视图可独立设置排序规则、显示字段与自定义 CSS 样式。

**操作日志审计**：记录所有新增、删除、修改与导出操作的详细日志，支持按操作者、时间区间、操作类型回溯，适用于团队协作场景。

## 应用场景

**技术文档撰写与维护**：开源项目维护者在撰写 README、用户手册或 API 文档时，需要引用大量外部规范、教程或示例。KnowledgeLink 可作为项目文档的补充资源库，集中管理所有外链，当外部链接变动时自动提醒，避免文档中出现死链。

**信息聚合与每日阅读**：产品经理、技术博主或行业研究员每日需浏览多个百科站点。使用本项目的分类标签与过滤功能，可快速筛选出当日更新的条目，生成个性化阅读列表，节省反复切换页面的时间。

**搜索引擎优化与内容推荐**：内容平台运营者可将收录的链接作为外部参考源，在文章底部或侧边栏展示“相关阅读”。通过元数据提取功能，可自动匹配当前文章主题与已有链接的关联度，实现精准推荐。

**学术文献参考整理**：研究人员在撰写论文或综述时，需整理大量网络参考文献。本项目的批量导入与导出功能可帮助快速构建参考文献清单，状态监控功能可确保提交前所有引用链接有效，避免审稿阶段出现链接访问错误。

## 快速开始

以下命令演示了从克隆代码库到启动本地服务的完整流程。请确保系统已安装 Git 与 Node.js（v18 及以上版本）。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/knowledgelink.git

# 进入项目根目录
cd knowledgelink

# 安装依赖项
npm install

# 启动开发服务器，默认监听 3000 端口
npm run dev
```

启动后，在浏览器中访问 `http://localhost:3000` 即可进入管理面板。首次运行将自动创建 SQLite 数据库文件并生成默认管理员账户，账户信息请查看控制台输出的初始化日志。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | v18.x 或 v20.x | 运行时环境，推荐使用 LTS 版本 |
| npm | v9.x 或 v10.x | 包管理器，随 Node.js 一同安装 |
| SQLite3 | v5.1.x（内嵌） | 默认轻量级数据库，无需额外安装 |
| Git | v2.30 或更高 | 用于克隆仓库和管理版本 |
| 操作系统 | Linux（Ubuntu 20.04+）、macOS（12+）或 Windows 10/11（WSL2 推荐） | 跨平台支持，生产环境建议 Linux |
| 内存 | 最低 512 MB，推荐 2 GB 以上 | 影响批量导入与状态监控任务并发度 |
| 磁盘空间 | 至少 1 GB | 用于存储数据库文件与日志 |
| 网络 | 稳定互联网连接 | 状态监控与元数据提取需访问外网 |

## 常见问题

**Q：状态监控任务占用过多带宽或 CPU，如何调优？**

A：可在 `config/default.json` 中调整 `monitor.interval`（监控间隔，单位分钟）与 `monitor.concurrency`（并发请求数）。默认间隔为 1440 分钟（每日一次），并发为 10。若服务器网络条件较差，建议将并发数降至 3-5，并延长间隔至 2880 分钟（每两日一次）。同时可启用 `monitor.timeout` 参数（单位毫秒）限制单次请求最长等待时间，默认 5000 ms。

**Q：导入包含大量 URL 的 CSV 文件时发生超时，如何解决？**

A：超时通常由于默认 HTTP 请求时限过短或导入事务过大。建议采用分块导入策略，将 CSV 拆分为每份 200-500 条记录的小文件分批导入。亦可修改 `config/default.json` 中的 `importer.batchSize` 与 `importer.timeout` 参数。若仍无法解决，可通过命令行工具 `node scripts/import-cli.js --file path/to/file.csv` 执行后台导入，该方式不受 Web 超时限制。

**Q：如何将 SQLite 数据库迁移至 PostgreSQL 用于生产环境？**

A：项目内置了迁移脚本 `scripts/migrate.js`，支持从 SQLite 导出数据并转换为 PostgreSQL 兼容格式。需先在 `config/production.json` 中配置 PostgreSQL 连接字符串（`db.url`），然后执行 `npm run migrate:pg`。迁移完成后，请手动更新 `config/production.json` 中的 `db.dialect` 为 `postgres`。建议在迁移前备份原始 SQLite 文件。
