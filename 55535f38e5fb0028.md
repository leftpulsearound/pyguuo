# LinkVault 技术资源索引

LinkVault 是一个面向开发者、技术研究人员与运维工程师的轻量级技术资源外链汇总与导航工具。该项目不存储任何实际内容，仅提供结构化、可检索的 URL 索引服务，帮助用户快速定位分散于互联网各处的技术文章、百科词条与工程实践文档。

本项目主要解决个人开发者与小型团队在技术调研过程中面临的资源分散、记忆负担重、检索效率低下等问题。通过将大量高质量外部链接按照主题与用途分类，并辅以本地化的标签与备注系统，LinkVault 可作为日常开发工作流中的知识入口层，显著降低信息查找的时间成本。

## 功能概览

**批量链接导入** 支持从纯文本文件、CSV 或 JSON 格式批量导入 URL 列表，自动去重并校验可访问性。

**分类与标签管理** 用户可为每条链接添加自定义分类标签与备注说明，支持多级分类体系，便于按技术栈、主题或项目阶段组织资源。

**全文检索与过滤** 基于 SQLite 全文索引引擎，提供对 URL、标题、标签及备注内容的快速检索能力，支持按分类、可用状态与最后验证时间过滤。

**可用性监控** 内置轻量级 HTTP 探测器，可定时检查已收录链接的可访问状态与响应码，自动标记失效链接并生成报告。

**导出与分享** 支持将选中的链接集合导出为 Markdown 列表、JSON 或 CSV 格式，便于团队内部分享或嵌入文档。

**RESTful API 接口** 提供标准 JSON API，允许第三方工具或脚本以编程方式查询、新增或更新链接记录，便于集成到自动化工作流中。

**本地 Web 仪表盘** 提供基于 Flask 的简易 Web 界面，用于浏览、搜索与管理链接资源，适合在本地或内网环境中部署使用。

## 应用场景

技术团队内部知识库建设。团队可将日常开发中遇到的有价值外部文章、官方文档、调试案例等统一收录至 LinkVault，并分配标签与责任人，形成可持续积累的团队知识资产。

个人开发者的技术阅读清单管理。开发者可利用 LinkVault 保存待读的技术博客、教程视频与工具官网，按学习路径或优先级分类，避免书签栏杂乱无章。

运维故障排查参考库。运维工程师可将常见的错误日志解决方案、系统调优参数说明、官方补丁公告等链接集中管理，配合可用性监控功能，确保关键参考资源始终可访问。

技术调研与选型文档辅助。在进行中间件选型或框架评估时，可将候选项目的官网、性能测试报告、社区讨论帖、安全公告等链接统一归档，便于横向对比与评审汇报。

## 快速开始

以下操作适用于 Linux / macOS 系统，Windows 用户可使用 WSL2 或 Git Bash 环境执行。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装 Python 依赖（推荐使用虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化 SQLite 数据库并导入示例链接
python scripts/init_db.py
python scripts/import_links.py --source data/sample_links.json

# 启动本地 Web 服务
python app.py --host 127.0.0.1 --port 8080
```

启动后，在浏览器中访问 http://127.0.0.1:8080 即可进入仪表盘。首次运行时会自动创建 admin 账户，初始密码输出在控制台日志中，请及时修改。

## 安装要求

| 依赖 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 及以上 | 核心运行环境，建议使用 3.11 或 3.12 长期支持版本 |
| SQLite | 3.35 及以上 | 内置于 Python 标准库，用于存储链接元数据与全文索引 |
| Flask | 2.2.x | Web 仪表盘与 REST API 的底层框架 |
| requests | 2.31.x | 用于可用性监控中的 HTTP 探测与响应码检查 |
| apscheduler | 3.10.x | 定时任务调度，用于周期性的链接可用性检查 |
| markdown | 3.5.x | 导出功能中生成 Markdown 格式列表的渲染器 |
| pytest | 7.4.x | 单元测试与集成测试框架（仅开发环境需要） |
| black | 24.x | 代码格式化工具（仅开发环境需要，用于保持代码风格统一） |

生产环境部署时，建议额外安装 gunicorn 作为 WSGI 服务器，并配置 Nginx 进行反向代理。数据库文件默认存储在 instance/links.db，可通过环境变量 DATABASE_PATH 自定义位置。

## 项目结构

│   ├── routes/                         # 路由蓝图
│   │   ├── web.py                      # 仪表盘页面路由 (渲染 HTML)
│   │   ├── api.py                      # RESTful API 端点实现
│   │   └── monitor.py                  # 可用性监控回调与状态更新
│   ├── models/                         # 数据模型与 ORM 映射
│   │   ├── link.py                     # Link 实体：URL、标题、标签、状态
│   │   ├── category.py                 # 分类树结构 (邻接表模型)
│   │   └── user.py                     # 用户账户与权限模型
│   ├── services/                       # 业务逻辑层
│   │   ├── importer.py                 # 批量导入服务 (CSV/JSON 解析)
│   │   ├── exporter.py                 # 导出服务 (Markdown/CSV/JSON)
│   │   ├── checker.py                  # HTTP 可用性检测器 (含重试与超时策略)
│   │   └── search.py                   # 全文检索引擎封装 (SQLite FTS5)
│   ├── static/                         # 前端静态资源
│   │   ├── css/                        # Tailwind CSS 定制样式

│       ├── index.html                  # 链接列表与搜索主页面
│       └── detail.html                 # 单条链接详情与编辑页

│   ├── import_links.py                 # 从外部文件批量导入链接
│   ├── export_links.py                 # 按条件导出链接集合

## 常见问题

**Q：LinkVault 会存储外部链接的镜像内容吗？**

A：不会。LinkVault 只存储 URL、标题、标签、分类与可用性状态等元数据，不抓取也不缓存外部页面的正文内容。所有链接在被访问时均直接跳转至原始来源，项目本身不涉及任何版权内容的存储与分发。

**Q：可用性监控是否会对外部站点造成压力？**

A：监控模块采用轻量级 HEAD 请求，且默认超时时间为 5 秒，间隔周期为 24 小时。每个链接在单次检查中仅发起一次请求，不会产生显著流量或并发压力。用户可在配置文件中调整检查间隔与请求方式（可切换为 GET 请求并禁用重定向）。

**Q：导入大量链接时性能如何？**

A：项目内置的批量导入器采用事务批量提交策略，单次导入 10000 条链接的耗时约为 3 至 5 秒（取决于磁盘 I/O 与数据校验逻辑）。全文索引更新采用增量模式，不会在导入过程中阻塞其他操作。建议单次导入规模控制在 5000 条以内，以获得最佳交互体验。
