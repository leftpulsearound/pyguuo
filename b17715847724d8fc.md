# ResourceBridge

ResourceBridge 是一个面向技术研究者、内容聚合者与知识管理者的外链资源归集与导航工具。该项目并非传统的爬虫或采集系统，而是一个基于人工筛选与结构化编排的静态资源索引库，旨在解决信息碎片化环境下高质量外链资源的发现、分类、存档与快速检索问题。

项目定位为技术资源的中转枢纽，适用于需要定期整理大量外部参考链接、百科条目、技术文档或行业资讯的个人知识工作者、开源社区文档维护小组以及企业技术中台的沉淀团队。ResourceBridge 不存储任何外部内容本身，仅提供链接的元数据描述与分类导航，完全尊重源站点的版权与访问规则。

## 功能概览

**批量链接导入** 支持从文本文件、CSV 或剪贴板批量导入原始 URL，自动去重并识别域名归属。

**自定义分类标签** 允许用户为每个链接添加多级标签，支持按主题、来源站点、文件类型或业务场景进行细粒度划分。

**元数据提取与补全** 自动请求目标页面的标题与描述信息，辅助用户快速了解链接内容，减少手动整理成本。

**全文检索与过滤** 基于标题、描述、标签和 URL 路径的轻量级全文搜索，配合多种过滤条件实现毫秒级定位。

**健康状态检查** 定时对已收录链接进行可达性探测，标记失效或重定向条目，保障资源库的长期可用性。

**静态站点导出** 将整理的资源库一键导出为静态 HTML 目录页，可直接托管至 GitHub Pages、Nginx 或对象存储服务，实现内部共享。

## 应用场景

**技术团队内部知识库构建** 开发团队在项目迭代过程中会积累大量参考文档、API 手册、故障排查案例与性能测试报告。ResourceBridge 可用于统一归集这些分散在各个浏览器书签、即时通讯群组或邮件中的链接，按服务模块或技术栈打标，新成员入职时可快速获得完整的上下文资源地图。

**开源项目文档的外链管理** 开源项目的 README 与 Wiki 中常常需要引用外部规范、教程或依赖项目。ResourceBridge 帮助维护者集中管理这些外链，当上游文档更新或站点迁移时，可批量检查并更新引用，避免文档中出现大量死链。

**行业资讯与研究素材归档** 分析师、研究员或产品经理需要持续跟踪特定领域的动态，每日会产生数十个待读或待归档的网页链接。ResourceBridge 提供轻量级的暂存与分类能力，配合标签体系，可按主题、时间或优先级生成个人阅读清单，提升信息处理效率。

## 快速开始

以下操作基于 Linux/macOS 环境，Windows 用户可使用 WSL 或 Git Bash 执行。

```bash
# 克隆项目仓库
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge

# 安装依赖（使用 Python 虚拟环境）
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 初始化本地数据库并启动开发服务
python manage.py migrate
python manage.py runserver 0.0.0.0:8000
```

服务启动后，访问 `http://localhost:8000` 即可进入资源管理面板。首次使用请根据引导创建管理员账户。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Python | 3.9 - 3.11 | 核心运行环境，推荐使用 3.10 长期支持版本 |
| PostgreSQL | 13.0 及以上 | 生产环境数据库，用于存储链接元数据与标签关系 |
| Redis | 6.2 及以上 | 缓存与任务队列后端，用于异步健康检查任务 |
| Node.js | 18.0 及以上 | 仅用于前端资源构建，后端运行无需 |
| Nginx | 1.20 及以上 | 生产环境反向代理与静态文件服务推荐 |
| Git | 2.30 及以上 | 版本控制与项目更新依赖 |

## 项目结构

```
resourcebridge/
├── manage.py                    # Django 项目管理入口
├── requirements.txt             # Python 依赖清单
├── .env.example                 # 环境变量模板（数据库/缓存/密钥）
├── config/                      # 项目级配置目录
│   ├── settings.py              # 基础配置（含分环境加载逻辑）
│   ├── settings_production.py   # 生产环境覆盖配置
│   └── urls.py                  # 根路由分发
├── apps/                        # 所有自定义应用
│   ├── core/                    # 核心数据模型（Link, Tag, Category）
│   │   ├── models.py            # 链接与标签的 ORM 定义
│   │   ├── admin.py             # 后台管理界面定制
│   │   └── services.py          # 链接去重、元数据抓取服务
│   ├── scanner/                 # 批量导入与健康检查模块
│   │   ├── tasks.py             # Celery 异步任务（检查链接状态）
│   │   ├── parsers.py           # CSV/JSON/纯文本导入解析器
│   │   └── checks.py            # HTTP 状态码与重定向检测逻辑
│   ├── search/                  # 搜索与过滤引擎
│   │   ├── indexes.py           # PostgreSQL 全文检索索引配置
│   │   └── filters.py           # 标签/域名/状态联合过滤
│   └── exporter/                # 静态站点生成器
│       ├── generators.py        # HTML 目录页与 RSS 生成
│       └── templates/           # 导出用的 Jinja2 模板
├── static/                      # 前端静态资源（CSS/JS/图标）
│   ├── css/                     # 基于 Tailwind 的定制样式
│   └── js/                      # 交互逻辑（搜索补全、标签管理）
├── templates/                   # Django 后台管理页面模板
├── docs/                        # 完整项目文档（详见文档导航）
├── scripts/                     # 运维与部署辅助脚本
│   ├── init_db.sh               # 数据库初始化
│   └── backup.sh                # 增量备份脚本
├── tests/                       # 单元测试与集成测试
│   ├── test_models.py
│   ├── test_scanner.py
│   └── test_api.py
├── docker-compose.yml           # 开发环境容器编排（PostgreSQL + Redis）
└── Dockerfile                   # 生产级容器镜像构建文件
```

## 常见问题

**Q：ResourceBridge 是否存储外部链接的实际内容或快照？**
A：不存储。该项目仅保存链接的 URL、标题、描述和用户自定义标签。所有对目标页面的访问均为实时请求，项目本身不充当代理或缓存服务器。用户应遵守目标站点的 robots.txt 与使用条款。

**Q：健康检查模块会频繁请求源站点吗？**
A：健康检查任务默认每 24 小时执行一次，且检查间隔时间可在配置文件中按域名单独设置。对于同一站点的批量链接，检查器会自动合并请求头并复用连接，避免对源服务器造成过大压力。用户可根据自身网络环境调整检查频率或完全禁用该功能。

**Q：如何将现有书签或收藏夹导入 ResourceBridge？**
A：项目内置了 CSV 与 JSON 导入接口。您可从主流浏览器导出书签为 HTML 文件，再通过在线工具或脚本转换为 CSV 格式（包含 URL 与标题列）。具体转换步骤请参考 docs/user-guide.md 中的「批量导入」章节。
