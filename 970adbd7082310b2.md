# LinkPedia

LinkPedia 是一个面向技术研究人员、开发者与内容策展人的外链资源归集与结构化导航系统。该项目不对任何外部内容进行二次编辑或转载，仅通过人工筛选与分类索引，将分散于网络各处的优质技术文档、百科条目、教程笔记与参考手册聚合为统一的访问入口。项目定位为“技术知识的外部索引簿”，旨在解决信息过载环境下高质量参考资源难以被发现、记忆与复用的问题。LinkPedia 适合作为技术团队内部的知识补给站、个人开发者的查资料辅助工具，以及技术写作人员的素材来源库。

## 功能概览

**多源聚合索引**：将来自同一主域下的海量文章条目按内部标识符进行归并，提供按发布时间、热度与分类的浏览视图。

**原始链接直出**：系统严格保留每条资源的原始 URL 字符串，不做协议升级、域名规范化或路径改写，确保访问路径与收录时完全一致。

**轻量级全文检索**：基于资源标题与摘要文本构建倒排索引，支持布尔查询与短语匹配，便于在大型收藏库中快速定位目标条目。

**分类标签体系**：每一条资源可附加多个主题标签，标签库覆盖编程语言、框架、算法、工程实践、运维监控等技术领域。

**收藏与批注功能**：用户可将常用资源加入个人收藏夹，并为每条资源添加私有备注，用于记录阅读心得或使用场景。

**导入导出机制**：支持以 JSON 和 CSV 格式批量导入/导出资源列表，便于与其他知识管理工具（如 Notion、Obsidian）进行数据交换。

## 应用场景

技术团队内部知识库构建：团队技术负责人可将 LinkPedia 作为团队文档系统的外链补充层，统一存放日常遇到的优秀技术博客、官方文档镜像和问题排查案例，避免知识碎片化。

个人开发者的学习路径管理：正在系统学习某一技术栈（如分布式系统、前端工程化）的开发者，可使用 LinkPedia 按主题收集相关深度文章，形成个人学习时间线上的阅读清单。

技术内容创作素材收集：技术博主或开源文档贡献者在撰写专题文章时，可通过 LinkPedia 快速整理需要引用的外部参考资料，并确保引用链接的可追溯性与原始性。

技术社区资源共建：开源社区或技术兴趣小组可利用 LinkPedia 的多用户协作能力，共同维护一份领域内的优质外链清单，降低新成员的信息搜寻成本。

## 快速开始

以下命令序列适用于 Linux/macOS 及 Windows WSL 环境，假定系统已安装 Git 与 Node.js（v18 及以上）。

```bash
# 克隆项目仓库至本地
git clone https://github.com/linkpedia/linkpedia-core.git
cd linkpedia-core

# 安装项目依赖（使用 npm）
npm install

# 以开发模式启动本地服务，默认监听端口 3000
npm run dev
```

启动成功后，访问控制台输出的本地地址即可进入 LinkPedia 的索引管理界面。首次启动将自动执行数据库迁移与示例数据加载。

## 贡献指南

1. 资源条目增删改：通过 Web 界面或 API 直接提交新资源 URL，系统将自动尝试抓取标题与描述。提交前请确认该资源尚未被收录，且内容属于技术类范畴。

2. 分类标签优化：若发现已有资源分类不当或标签缺失，可编辑资源详情页的标签字段。标签命名应遵循短横线命名法，如 `distributed-system`、`react-hooks`。

3. 代码贡献：所有功能开发与缺陷修复均基于 GitHub Flow。请从 `main` 分支派生新功能分支，完成开发后提交 Pull Request，并在描述中注明关联 Issue 编号。

4. 文档完善：文档位于 `/docs` 目录，采用 Markdown 格式编写。若发现文档错误或遗漏，可直接修改对应文件并提交 Pull Request。

5. 问题反馈：使用 GitHub Issues 报告 Bug 或提出功能建议，提交时请附上详细的复现步骤、环境信息以及日志片段（若适用）。

## 常见问题

Q: 为什么资源列表中的 URL 不做任何规范化处理，即使访问失败也不会自动修正？

A: LinkPedia 的设计原则是“忠实记录原始引用”。我们不对收录的 URL 进行任何语义改写，包括协议升级、域名标准化或路径大小写转换。这一策略确保了数据来源的可审计性，也避免因自动修正引入新的访问错误。如果某条链接失效，用户可通过系统内的“报告失效链接”功能进行标记，由维护者人工核实后处理。

Q: 项目是否提供对外公开的 API 接口，以便与其他工具集成？

A: 是的。LinkPedia 在 `/api/v1` 路径下提供了 RESTful 风格的公开 API，支持资源的查询、新增、更新和删除操作，以及标签管理功能。API 请求需附带 Bearer Token 进行身份验证，Token 可在用户设置页面生成。完整的 API 文档位于 `/docs/developer-guide/api-reference.md`。

Q: 如何迁移现有收藏夹或书签数据到 LinkPedia？

A: 项目支持通过 CSV 或 JSON 格式批量导入资源列表。导入前请参考 `/docs/data-spec/import-format.md` 中的字段定义，将现有数据转换为符合规范的格式后，在管理后台的“批量导入”界面提交文件。导入过程为异步任务，处理完成后会通过站内通知告知结果。
