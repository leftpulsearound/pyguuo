# LinkVault 技术资源索引系统

LinkVault 是一个面向开发者、技术研究人员及内容策展人的轻量级技术资源外链汇总与导航系统。该项目旨在解决个人或团队在浏览、收集、整理海量分散技术文章、百科条目及开发文档时面临的检索效率低下、链接散落、上下文缺失等问题。通过结构化的数据组织与静态站点生成机制，LinkVault 将原始 URL 列表转化为可分类、可检索、可分享的技术知识库，适用于搭建个人技术书架、团队知识周报或项目外部参考库。

## 功能概览

**批量链接导入**：支持通过文本文件或标准输入流一次性导入大批量 URL，自动解析并去重，适用于批次处理场景。

**自动元数据抓取**：对导入的每个链接尝试拉取页面标题、概要描述及最后修改时间，为后续检索提供基础数据。

**自定义标签与分类**：用户可为每条记录添加多级标签与分类标记，实现按技术领域、项目阶段或优先级的多维度筛选。

**全文搜索与过滤**：基于标题、描述及标签构建的轻量级倒排索引，支持快速全文检索与多条件组合过滤。

**静态站点生成**：一键将索引数据导出为静态 HTML 目录页，包含分类导航、标签云和链接卡片视图，适合部署至任意 Web 服务器。

**数据导入导出**：支持 JSON、CSV、Markdown 表格三种格式的导入导出，便于与其它工具链（如 Excel、Notion、Obsidian）进行数据交换。

## 应用场景

**技术团队周报汇总**：团队技术负责人每周将组内成员分享的参考链接（如性能优化案例、新框架发布公告、Bug 分析文章）批量导入 LinkVault，自动生成带摘要的周报链接清单，发送给全体开发人员查阅。

**开源项目外部参考库**：开源项目维护者将项目依赖的协议解读、底层原理剖析、相似方案对比等外部资料统一收录至 LinkVault，作为项目 Wiki 的补充外链索引，帮助新贡献者快速理解设计决策背景。

**个人技术阅读工作流**：开发者每天浏览技术社区时，将值得回读的文章链接暂存至 LinkVault 的待读队列，利用周末时间统一查阅，查阅后添加学习笔记标签，形成个人长期技术知识图谱。

**技术文档迁移辅助**：在系统重构或文档平台切换期间，使用 LinkVault 对旧文档站点中的所有外链进行快照式汇总，生成迁移前后的链接对照表，有效避免死链与内容丢失。

## 快速开始

以下命令序列可在任意 Linux/macOS 或 Windows WSL 环境中完成 LinkVault 的克隆、依赖安装与初次运行。

```bash
# 克隆项目仓库至本地
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装 Python 依赖（建议使用虚拟环境）
python3 -m venv venv
source venv/bin/activate   # Windows 下使用 venv\Scripts\activate
pip install -r requirements.txt

# 执行首次运行，导入示例链接并生成静态站点
python linkvault.py --import samples/links.txt --output dist/
```

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Python | 3.8 及以上 | 核心运行环境，用于解析 URL、抓取元数据及生成静态页面 |
| pip | 20.0 及以上 | Python 包管理工具，用于安装依赖库 |
| requests | 2.25.0 及以上 | 发送 HTTP 请求以获取链接页面的标题与 Meta 信息 |

| jinja2 | 3.0.0 及以上 | 模板引擎，用于将索引数据渲染为静态 HTML 页面 |
| tldextract | 3.1.0 及以上 | 从 URL 中准确提取域名、子域名及后缀，辅助分类统计 |

## 项目结构

项目采用分层模块化设计，核心逻辑与辅助工具分离，便于扩展与维护。

```
linkvault/
├── linkvault.py                # 命令行入口，处理参数解析与流程编排
├── requirements.txt            # 生产环境依赖清单
├── README.md                   # 项目说明文档（即本文档）
├── LICENSE                     # MIT 许可证文本
├── config/
│   ├── default.yaml            # 默认配置项：站点标题、输出路径、请求超时等
│   └── schema.json             # 导入数据格式的 JSON Schema 校验文件
├── core/
│   ├── __init__.py
│   ├── importer.py             # 链接导入模块：支持文本、CSV、Markdown 表格解析
│   ├── crawler.py              # 元数据抓取模块：含重试策略与 User-Agent 轮换
│   ├── indexer.py              # 索引构建模块：管理标签、分类与倒排表
│   └── exporter.py             # 导出模块：输出 JSON/CSV/静态 HTML
├── templates/
│   ├── base.html               # 基础 HTML 模板，定义全局骨架与 CSS 变量
│   ├── index.html              # 首页模板：展示分类导航与最新添加链接
│   └── detail.html             # 详情页模板：单条链接的完整元数据与标签
├── static/
│   ├── style.css               # 响应式布局样式，适配桌面与移动端
│   └── search.js               # 前端搜索交互脚本，实现即时过滤与高亮
├── tests/
│   ├── test_importer.py        # 导入器单元测试，覆盖各类输入格式
│   ├── test_crawler.py         # 抓取器测试，含 Mock HTTP 响应
│   └── fixtures/
│       └── sample_links.txt    # 测试用的示例链接列表
└── dist/                       # 默认输出目录（git ignored），存放生成的静态站点
    ├── index.html
    └── articles/
        └── *.html
```

## 常见问题

Q: 导入大量 URL 时，程序因网络超时或连接中断而停止，如何恢复进度？

A: LinkVault 在导入过程中会生成临时缓存文件 import_cache.json，记录已成功抓取元数据的 URL。重新运行导入命令时，程序会自动跳过缓存中已有的记录，仅处理新链接或之前失败的条目。您也可以通过 --retry-failed 参数强制重试所有失败记录。

Q: 生成的静态页面中部分链接标题显示为 URL 本身，未正确抓取到页面标题，是什么原因？

A: 这通常是因为目标服务器对爬虫请求返回了非 200 状态码，或页面使用了 JavaScript 动态渲染标题。您可以尝试调整 config/default.yaml 中的 user_agent 列表，更换为常见浏览器标识；对于 JS 渲染严重的站点，建议手动在导入的 CSV 文件中预先填写 title 列，系统将优先使用用户提供的值。

Q: 如何将 LinkVault 部署为团队内部持续更新的链接数据库，而不是每次手动运行？

A: 推荐使用 cron 或 systemd timer 设置定期任务（例如每周一上午 9:00），执行 linkvault.py --update 命令，该模式会增量拉取预设源列表中的新链接并重新生成静态页面。同时，可将 dist/ 目录挂载至 Nginx 或 S3 存储桶，实现内容自动刷新。
