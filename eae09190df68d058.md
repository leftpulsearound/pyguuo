# LinkVault 知识索引系统

LinkVault 是一个面向技术研究者、文档维护者和信息聚合场景的轻量级外链资源汇总与导航工具。该项目并非传统的内容管理系统，而是一个结构化的外部知识索引库，旨在帮助团队或个人将散落在各处的优质百科类文章、技术文档与参考链接进行统一收纳、分类标注与快速检索。

项目目标用户包括技术文档工程师、开源项目维护者、学术研究者以及任何需要长期管理大量外链资源的知识工作者。LinkVault 不存储文章正文，只保留链接元数据与自定义标签体系，从而避免内容合规风险并降低存储成本。通过本项目提供的目录模板与自动化校验脚本，用户可以在一小时内完成从零到一的资源索引搭建，并持续维护一套可私有化部署的导航站点。

## 功能概览

**批量链接导入** 支持通过 CSV 或 JSON 格式批量录入外部 URL，自动解析域名并生成初步分类建议。

**自定义标签体系** 允许用户为每条链接附加多个层级标签，如“技术栈-后端-数据库”或“领域-医学-临床”，满足多维度筛选需求。

**链接可用性巡检** 内置定时任务模块，可周期性检测已收录链接的 HTTP 状态码，自动标记失效或重定向的条目。

**全文模糊检索** 基于标题、标签和自定义备注字段提供轻量级全文搜索，支持拼音首字母快速定位。

**多格式数据导出** 支持将索引数据导出为 Markdown 表格、HTML 导航页或纯文本清单，便于嵌入其他文档系统。

**访问统计看板** 记录每条链接的点击次数与最近访问时间，辅助用户识别高频使用的核心资源。

**权限分级管理** 提供管理员、编辑者、只读访客三级角色控制，适合团队协作场景下的链接库维护。

## 应用场景

技术团队内部知识库建设。开发团队在项目迭代过程中会积累大量第三方依赖文档、故障排查帖子和官方 API 参考，LinkVault 可将这些分散的浏览器书签统一入库，按服务模块打标，新成员入职时可直接按标签筛选学习资料。

学术文献与数据来源管理。研究人员在撰写综述或开展课题时，需要记录数百个统计数据来源、疾病百科词条和临床试验注册号，LinkVault 的备注字段可记录引用日期与摘要，替代传统的 Excel 手工管理方式。

开源项目 README 外链维护。开源项目的文档中常包含大量“参考资源”或“进一步阅读”链接，LinkVault 提供链接变更监控功能，当某个参考链接永久移动时自动提醒维护者更新 README，避免文档腐化。

个人知识体系构建。独立博主或在线教育者可将多年积累的教程、案例和工具网站通过 LinkVault 结构化存储，并利用导出功能生成静态导航页面，分享给读者或学员。

## 快速开始

以下命令适用于 Linux / macOS / Windows WSL 环境，需提前安装 Git 与 Node.js 18.x 以上版本。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/linkvault.git
cd linkvault

# 安装项目依赖
npm install

# 以开发模式启动本地服务
npm run dev
```

执行完毕后，打开浏览器访问 http://localhost:5173 即可进入索引管理界面。首次启动会自动创建示例数据文件，包含约 20 条预置百科链接供体验。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---|---|---|
| Node.js | 18.0.0 及以上 | 运行时环境，推荐使用 LTS 版本 |
| npm | 9.0.0 及以上 | 包管理器，用于安装项目依赖 |
| SQLite | 3.35.0 及以上 | 嵌入式数据库，用于存储链接元数据与标签 |
| Git | 2.30.0 及以上 | 版本控制工具，用于克隆和提交变更 |
| 操作系统 | Linux / macOS / Windows 10+ | 支持 WSL 环境，生产部署推荐 Linux 内核 4.0+ |
| 浏览器 | Chrome 90+ / Firefox 88+ / Edge 90+ | 管理界面访问需要现代浏览器支持 |
| 内存 | 最低 512MB，推荐 1GB | 索引 10 万条以下链接时内存占用小于 256MB |
| 存储 | 最低 100MB 可用空间 | 每万条链接约占用 15MB 存储（含索引） |

## 项目结构

```
linkvault/
├── src/                           # 核心源代码目录
│   ├── main.ts                    # 应用入口，初始化服务器与数据库连接
│   ├── routes/                    # 路由定义层，处理 HTTP 请求分发
│   │   ├── links.ts               # 链接增删改查及导入导出路由
│   │   ├── tags.ts                # 标签管理路由
│   │   └── health.ts              # 巡检任务触发与状态查询路由
│   ├── services/                  # 业务逻辑层，封装核心功能
│   │   ├── linkService.ts         # 链接索引的查重、校验与存储逻辑
│   │   ├── tagService.ts          # 标签合并、重命名与层级管理
│   │   └── checkerService.ts      # 定时巡检调度器与 HTTP 状态检测
│   ├── repositories/              # 数据访问层，封装 SQLite 操作
│   │   ├── linkRepository.ts      # 链接表的 CRUD 与全文搜索查询
│   │   └── tagRepository.ts       # 标签表的关联查询与统计
│   ├── models/                    # 数据模型定义（TypeScript 接口与 Zod 校验）
│   │   ├── link.ts                # 链接实体字段与校验规则
│   │   └── tag.ts                 # 标签实体与层级约束
│   └── utils/                     # 通用工具函数
│       ├── validator.ts           # URL 格式校验与域名黑名单过滤
│       └── exporter.ts            # Markdown / HTML / CSV 导出生成器
```

## 贡献指南

我们欢迎并感谢任何形式的贡献，包括但不限于功能建议、文档改进、缺陷报告和代码提交。请遵循以下流程以确保协作顺畅。

第一步：查阅问题追踪器。访问 GitHub Issues 页面，查找标记为“help wanted”或“good first issue”的待办事项，避免重复工作。如需新增功能，请先创建一个 Issue 描述需求场景与预期行为。

第二步：派生项目并创建分支。将主仓库 Fork 至个人账户，然后克隆到本地。请基于 main 分支创建一个具有描述性的新分支，命名格式为 `feature/功能简述` 或 `fix/问题编号`。

第三步：编写代码并添加测试。所有新增功能必须包含对应的单元测试或集成测试，确保测试覆盖率达到 80% 以上。代码风格遵循项目配置的 ESLint 与 Prettier 规则，提交前执行 `npm run lint` 与 `npm run test` 进行自检。

第四步：提交变更并签署开发者原创声明。提交信息需遵循 Conventional Commits 规范，格式为 `<type>: <subject>`，例如 `feat: add batch import progress bar`。提交时默认同意将代码以 MIT 许可证进行分发。

第五步：发起拉取请求。推送到派生仓库后，在主仓库发起 Pull Request，并填写 PR 模板中的检查清单。至少需要一位维护者审核通过后方可合并。若有冲突，请及时基于最新 main 分支进行变基操作。

## 常见问题

问：导入大量链接时页面卡顿或超时怎么办？
答：单次导入建议不超过 5000 条。若需批量导入上万条链接，请使用 CLI 模式下的 `npm run import:bulk -- --file=./data/links.json` 命令，该命令会绕过 HTTP 超时限制并在后台异步处理。同时建议将 Node.js 内存上限调整为 2GB：`NODE_OPTIONS="--max-old-space-size=2048" npm run import:bulk`。

问：巡检模块检测到失效链接后如何自动处理？
答：默认配置下，系统仅标记状态为“失效”并发送日志警告，不会自动删除。您可以在 `config/default.yaml` 中将 `autoArchive` 参数设为 `true`，此时连续三次检测失败的链接将被移至归档表，不再出现在主列表中。如需恢复，可通过管理界面的“回收站”功能还原。

问：如何将现有浏览器书签（Chrome / Firefox）快速导入 LinkVault？
答：请先从浏览器导出书签为 HTML 文件，然后使用项目提供的转换脚本：`npm run convert:bookmark -- --input=./bookmarks.html --output=./data/import.json`。该脚本会自动解析书签文件夹层级并将其映射为标签结构，随后通过标准导入接口完成入库。注意嵌套层级超过 5 级时可能产生冗余标签，建议导入前适当扁平化。
