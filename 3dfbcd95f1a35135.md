# ResourceIndex

一个面向开发者的技术资源导航与外链聚合工具。该项目旨在解决开发者面临的信息碎片化问题，通过人工筛选与分类索引，将散落于网络各处的优质技术文章、百科词条、开发文档及实用工具集中呈现。ResourceIndex 并非搜索引擎，而是一个经过整理的、结构化的知识入口，适用于希望提升信息获取效率的软件工程师、运维人员、技术研究者及学生。

项目采用静态站点生成机制，所有外链资源按主题与用途进行标签化组织。用户可通过本地克隆与简单配置，快速搭建属于个人或团队的内部资源导航站。项目核心价值在于减少重复检索时间，提供可维护、可扩展的参考信息基线。

## 功能概览

**批量链接导入与解析**：支持从 CSV、JSON 或纯文本列表批量导入 URL，自动提取标题与元描述，生成初始索引条目。

**多维度分类与标签系统**：每条资源可关联多个分类与自定义标签，支持按技术领域、难度等级、资源类型进行筛选与排序。

**本地全文检索**：基于 MiniSearch 或 Lunr.js 实现轻量级客户端全文搜索，无需依赖外部搜索引擎即可对标题、描述及标签进行快速匹配。

**静态站点生成**：内置模板引擎，可将索引数据渲染为纯静态 HTML 页面，便于部署至 GitHub Pages、Nginx 或任何 Web 服务器。

**数据快照与版本管理**：每次更新自动生成数据快照（JSON 格式），支持回滚与变更对比，适合团队协作维护资源库。

**可配置的抓取规则**：允许用户定义爬取深度、请求间隔及 User-Agent，针对特定站点定制采集策略，降低被屏蔽风险。

**导入导出兼容性**：支持导出为 Markdown 表格、CSV 或 JSON 格式，便于迁移至其他知识管理工具或进行二次数据分析。

**响应式管理面板**：提供基于 Bootstrap 的后台界面，在移动端与桌面端均可完成资源增删改查操作。

## 应用场景

技术团队内部知识库建设：团队可将项目部署在内网服务器，用于沉淀项目开发中遇到的踩坑记录、官方文档镜像链接以及内部规范文档，新成员入职时可快速获取所需资料。

个人开发者的阅读清单管理：开发者可将日常浏览发现的优质博客、教程及 API 参考收录至 ResourceIndex，按学习路径（如 Go 语言基础、并发编程、微服务）组织，形成个人成长知识图谱。

开源项目文档聚合：开源社区维护者可利用本工具整理与项目相关的周边生态链接（如插件列表、部署教程、性能测试报告），作为官方文档的补充导航页。

技术培训机构的教学辅助：讲师可将课程涉及的预习资料、扩展阅读及实验环境入口集中配置于 ResourceIndex，学员通过单一地址即可访问所有教学资源，避免多平台跳转。

## 快速开始

以下命令将在当前目录下克隆项目、安装依赖并启动开发服务器。

```bash
git clone https://github.com/resource-index/resourceindex.git
cd resourceindex
npm install
npm run dev
```

执行成功后，访问控制台输出的本地地址（默认为 http://localhost:3000）即可进入管理界面。首次启动将自动生成示例数据，包含预置分类与占位资源。

## 项目结构

```
resourceindex/
├── config/                           # 项目配置文件目录
│   ├── default.json                  # 默认环境变量（端口、数据库路径、缓存策略）
│   └── crawler-rules.json            # 针对不同站点的抓取频率与超时设置
├── src/
│   ├── core/                         # 核心业务逻辑层
│   │   ├── indexer.js                # 资源索引增删改查及标签关联逻辑
│   │   ├── parser.js                 # 从外部 URL 提取标题、描述与关键词
│   │   └── search.js                 # 本地全文搜索引擎封装
│   ├── routes/                       # HTTP 路由层（Express/Koa）
│   │   ├── api.js                    # RESTful API 端点（/api/resources, /api/tags）
│   │   └── web.js                    # 管理后台页面路由及静态页面渲染
│   ├── models/                       # 数据模型层（SQLite ORM）
│   │   ├── Resource.js               # 资源条目模型，包含 URL、标题、摘要、分类外键
│   │   ├── Category.js               # 分类模型，支持无限层级嵌套
│   │   └── Snapshot.js               # 数据快照模型，用于版本回滚
│   ├── services/                     # 外部服务集成
│   │   ├── fetcher.js                # 基于 axios 的 HTTP 请求封装，含重试与代理配置
│   │   └── exporter.js               # 导出为 CSV/JSON/Markdown 的格式化工具
│   ├── ui/                           # 前端资源（管理面板）
│   │   ├── static/                   # CSS、JS、图片等静态资产
│   │   ├── templates/                # Nunjucks/EJS 模板文件
│   │   └── components/               # 可复用的 Vue/React 组件（可选）
│   └── cli/                          # 命令行工具入口
│       ├── import.js                 # 从本地文件批量导入链接列表
```

## 常见问题

如何迁移已有书签或收藏夹数据到本工具？
项目提供了导入脚本 cli/import.js，支持 Netscape HTML 书签导出格式（所有主流浏览器均支持）。您也可以将 CSV 文件按示例格式（列：url, title, description, tags）组织后通过 `npm run import -- --file=./bookmarks.csv` 导入。

搜索时某些链接无法被检索到，可能是什么原因？
请检查该资源条目是否包含有效的标题与描述字段。若标题为空，搜索索引将跳过该条目。您也可以通过管理后台手动编辑补充缺失信息。此外，默认搜索仅匹配标题、标签和分类名称，不匹配正文内容。

如何定期自动更新外部链接的有效性？
项目内置了链接检查服务，可通过 `npm run check -- --interval=weekly` 启动定期检查任务。该任务会发送 HEAD 请求验证 URL 是否可访问，并在管理面板的“异常链接”视图中展示失效条目。您也可以配置 webhook 在发现大量失效链接时发送邮件通知。
