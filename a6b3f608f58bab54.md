# Vokav Knowledge Bridge

Vokav Knowledge Bridge 是一个面向技术研究者、内容聚合者与知识管理工程师的轻量级外链资源归集与结构化导航系统。该项目定位于将分散在多个内容源头的百科类、技术类、资讯类文章通过统一的条目索引机制进行整理，形成可被程序化读取、检索与引用的只读知识图谱。

项目不提供内容托管或改写服务，而是以严格的外链原样收录机制，构建一个可审计、可追溯、高可用的参考链接仓库。目标用户包括文档工程师、技术写作团队、自动化测试脚本维护者、爬虫策略设计人员以及需要批量校验存量链接可用性的运维工程师。

## 功能概览

**批量链接原样收录** 支持一次性导入大量外部 URL，系统严格保留协议、域名、路径、大小写及文件扩展名，不进行任何自动补全或规范化改写。

**链接分类与标签化索引** 允许为每条收录链接分配类别标签与简要描述，便于按主题域或内容类型进行快速筛选。

**结构化的文档导航** 内置多层级目录树与表格化视图，帮助维护者理解链接的组织逻辑与查询路径。

**只读镜像访问模式** 所有外链资源仅作为只读引用源，项目本身不缓存页面内容，不转发请求，不修改目标地址。

**自动链接状态检测接口** 提供可选的可用性检查脚本，支持定时探测收录链接的 HTTP 状态码并生成报告。

**Markdown 原生配置格式** 所有资源列表、导航结构、依赖说明均采用纯 Markdown 编写，便于版本管理与差异对比。

## 应用场景

自动化文档链接触发器维护 技术写作团队可将该项目作为外部引用来源的统一登记处，每次新增或变更外链时，只需更新项目中的资源列表文件，CI 流程即可自动生成最新的链接校验报告。

爬虫策略的入口白名单管理 数据采集工程师可以将收录的 URL 作为合法抓取目标的初始种子集合，结合项目提供的状态检测工具，定期过滤失效来源，保持爬虫列表的清洁性。

开源项目 README 的外链引用池 开源项目维护者可在自己的 README 中引用本项目作为“参考来源索引”，避免在多个仓库中重复维护相同的链接集合，实现引用资源的单一事实源。

知识库迁移前的链接摸底 在将旧版百科系统或内容管理系统迁移至新平台前，运维人员可通过本项目收录所有历史文章链接，批量检测其可访问性，评估迁移后重定向策略的覆盖范围。

## 快速开始

```bash
# 克隆项目仓库
git clone https://github.com/vokav/knowledge-bridge.git

# 进入项目目录
cd knowledge-bridge

# 安装依赖（基于 Python 3.10+ 与 pip）
pip install -r requirements.txt

# 运行链接状态检测示例脚本
python scripts/check_links.py --input resources/links.txt --output reports/status.csv
```

## 贡献指南

提交新增链接的 Pull Request 时，请将原始 URL 追加到 resources/links.txt 文件末尾，并确保每一行仅包含一个完整的 URL，不添加额外注释或空格。PR 描述中应说明该链接所属的主题类别及预期用途。

若需要调整链接分类或标签结构，请修改 resources/categories.yaml 文件，并同步更新 docs/maintenance.md 中对应的分类说明。分类键名应使用小写字母与连字符，避免使用特殊符号。

运行本地链接检测脚本并通过所有测试用例。在提交前执行 python scripts/check_links.py --input resources/links.txt --timeout 5，确保新增链接的响应状态码位于 200-299 范围内，若为 301 或 302 需在备注中说明重定向目标。

更新 docs/examples.md 中的相关示例以反映新增资源的用法，若新增链接属于新的内容域，则需在 docs/usage.md 中补充查询或过滤该域的说明语句。

所有提交必须附带清晰的 commit message，格式为 type(scope): subject，其中 type 可取 add、fix、docs、chore，scope 为 links、config、scripts 或 docs。

## 常见问题

如何批量导入外部已有的链接集合

项目接受通过标准输入或文件复制的方式追加链接。您可以将现有链接列表按行合并至 resources/links.txt，随后运行链接检测脚本以验证其可用性。项目不提供自动去重功能，维护者需在合并前自行清理重复条目。

检测脚本报告大量超时或连接拒绝如何处理
