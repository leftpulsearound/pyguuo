# WapKnowledge Indexer

WapKnowledge Indexer 是一个面向移动端知识库与百科类内容的结构化索引与导航系统。该项目定位于技术信息聚合与资源外链管理，主要服务于需要对分散的百科条目、技术文档、历史资料及常见问答进行统一归集与快速检索的开发者和内容研究者。通过将大量半结构化的知识页面转化为可机器解析、可分类浏览的索引体系，WapKnowledge Indexer 有效降低了信息碎片化带来的查找与维护成本。

本项目并非一个传统的动态网站或内容管理系统，而是一个静态化的外链汇总与元数据标注工具。它接受一批原始百科类 URL，通过可配置的解析规则提取页面标题、分类标签、摘要及关联条目，最终生成结构化的导航页面或 API 输出。项目核心受众包括个人知识库维护者、技术文档工程师、数据采集与清洗人员以及轻量级内容站点的运营者。

## 功能概览

**批量 URL 导入与校验**：支持从文本文件或标准输入中批量读取 URL 列表，自动执行协议一致性检查与去重操作，并提供导入日志记录。

**多维度分类索引**：根据页面路径中的分类标识、数字 ID 段或自定义规则，为每条资源自动分配所属知识域，并生成分类树视图。

**元数据缓存与更新**：对已解析的页面标题、摘要及关键词进行本地缓存，支持增量更新策略，避免重复抓取与解析。

**结构化输出生成**：可将索引结果输出为 Markdown 导航页、JSON API 响应或 CSV 汇总表，适配不同的前端展示与数据处理需求。

**黑名单与优先级标记**：支持用户自定义黑名单规则过滤低质量或无关条目，同时支持手动标记高优先级资源以便置顶展示。

**查询过滤与全文搜索**：内置简单的关键词过滤接口，支持按标题、分类、ID 范围进行筛选，便于快速定位特定资源集合。

## 应用场景

**技术文档库的快速导航构建**：当团队拥有大量分散在 Wiki 或知识库平台中的技术文档时，可使用 WapKnowledge Indexer 导入现有页面链接，自动生成按模块或产品线组织的索引目录，替代手动维护的文档清单。

**百科类数据的轻量化迁移辅助**：在进行站点迁移或内容整合时，通过导入原始百科页面的 URL 集合，快速获取页面元数据并生成迁移清单，帮助评估内容规模与优先级。

**个人知识管理（PKM）系统的外部源接入**：对于使用 Obsidian、Logseq 等工具的用户，可将本项目作为外部插件的数据源，将远程百科条目索引后导入本地知识库，形成内外部知识联动。

**教育资源汇总站的资源盘点**：教育机构或在线学习平台可利用本项目对课程参考资料、扩展阅读链接进行定期盘点，标注失效链接与新增资源，维护资源库的健康度。

## 快速开始

以下步骤将指导您在本地环境中快速启动 WapKnowledge Indexer 并导入示例资源列表。

```bash
# 克隆项目仓库至本地
git clone https://github.com/your-org/wapknowledge-indexer.git

# 进入项目根目录
cd wapknowledge-indexer

# 安装项目依赖（使用 pip 或 poetry）
pip install -r requirements.txt

# 或使用 poetry
poetry install

# 运行索引构建脚本，导入位于 data/urls.txt 中的 URL 列表
python indexer.py build --input data/urls.txt --output output/index.json

# 生成 Markdown 导航页面
python indexer.py render --input output/index.json --output docs/navigation.md
```

## 项目结构

```
wapknowledge-indexer/

├── requirements.txt               # 生产环境依赖列表
├── pyproject.toml                 # 项目元数据与构建配置（Poetry）
├── config/
│   ├── default.yaml               # 默认解析规则与分类映射配置
│   └── custom_rules.json          # 用户自定义规则示例文件
├── indexer/
│   ├── __init__.py                # 核心模块初始化
│   ├── fetcher.py                 # HTTP 请求与页面获取模块
│   ├── parser.py                  # HTML 元数据提取与清洗逻辑
│   ├── classifier.py              # 基于路径与关键词的分类器
│   ├── cache.py                   # 本地缓存管理（SQLite 后端）
│   └── exporter.py                # JSON / Markdown / CSV 输出生成器
```

## 常见问题

**Q: 索引器能否处理需要登录或验证的页面？**

A: 当前版本仅支持公开可访问的页面。对于需要 Cookie 或 Token 验证的资源，您可以通过在 config/default.yaml 中配置自定义请求头（headers）字段来实现基础的身份传递，但本项目不提供完整的会话管理或交互式登录功能。建议在导入前使用外部工具将受保护页面下载为静态 HTML 并放置于本地路径。

**Q: 如何更新已导入资源的元数据，而不重新处理全部 URL？**

A: 索引器默认启用缓存机制。首次构建时，所有元数据将保存至 output/cache.db 文件中。当您再次运行 build 命令并传入相同的 URL 列表时，系统会优先检查缓存记录的时间戳。您可以通过添加 --force-update 参数强制忽略缓存并重新抓取所有页面。若仅需更新部分资源，请编辑 data/urls.txt 文件，保留待更新条目并删除其余行，然后执行增量构建。

**Q: 输出 JSON 中每个条目的字段结构是怎样的？**

A: 每个资源条目包含以下标准字段：id（数字标识）、url（原始链接）、title（从页面提取的标题）、category（自动分类标签）、summary（首段摘要或前 200 个字符）、last_updated（ISO 格式时间戳）以及 status_code（HTTP 响应状态）。完整 schema 定义可参考 docs/api/output-schema.json，您也可以使用 --schema-only 参数生成一个空模板用于二次开发。
